# 言語サーバ

`ss lsp` は標準入力と標準出力で動きます．VS Code 拡張は `ss.cli.path` で指定された実行ファイルに `lsp` 引数を付けて起動します．

```sh
ss lsp
```

通常はエディタが起動します．直接実行するのは，LSP 通信を調べる場合です．

## 機能

言語サーバは次の機能を返します．

| 機能 | エディタでの用途 |
| --- | --- |
| テキスト同期 | 編集後のプロジェクトスナップショット再構築 |
| 補完 | 関数，変数，宣言，フィールド |
| hover | 関数の型，変数の型，クラスやフィールド情報 |
| 定義ジャンプ | 解析中に記録した定義位置への移動 |
| インレイヒント | 引数名と推定型 |
| 文書シンボル | ページ，関数，定数，宣言の一覧 |
| 折りたたみ範囲 | 折りたたみ可能なブロック |
| semantic token | キーワード，関数，変数，文字列，数値，型，プロパティ |
| 色 | 色リテラルと色の表示候補 |

言語サーバの情報は，コンパイラの処理で作る editor metadata から来ます．

## スナップショット

編集されたファイルごとに，言語サーバはプロジェクト情報を解決し，ソースを読み，構文解析，import 読み込み，初期 IR 構築，検査を行い，スナップショットを保存します．

```text
text document -> project resolution -> source graph -> checked IR -> editor metadata
```

スナップショットには，入口パス，アセット基準，ローカルモジュール，診断，定義，変数，シンボル，ヒントが含まれます．

## projectInfo 要求

拡張はライブプレビューのために `ss/projectInfo` という独自要求を使います．返り値には次の情報が入ります．

```json
{
  "entryPath": "/path/to/slide.ss",
  "assetBaseDir": "/path/to/assets",
  "localModules": ["/path/to/helpers.ss"]
}
```

ライブプレビューはこのパスを使ってスナップショットを作り，`ss render` を呼びます．

## trace

プロトコルメッセージを見る場合は，`ss.lsp.trace.server` を設定します．

```json
{
  "ss.lsp.trace.server": "messages"
}
```

メッセージ本文まで必要な場合は `verbose` を使います．trace 出力は `ss` 出力チャンネルに出ます．

## CLI との一致

同じバイナリと同じプロジェクトルートを使う場合，エディタ診断は `ss check` と一致するはずです．

```sh
ss check --project .
```

違いがある場合は，`ss.cli.path`，ワークスペースフォルダ，最寄りの `ss.toml` を比較します．
