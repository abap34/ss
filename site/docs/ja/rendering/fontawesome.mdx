# Font Awesome

Markdown の画像記法に Font Awesome の参照を渡すと，アイコンを文字列中に配置できます．

## 描画の位置付け

描画は，構文解析，読み込み，型検査，展開，正規化，配置解決の後に行われます．

```text
source -> typed program -> document terms -> core graph -> layout -> PDF
```

描画器は，core graph に含まれる object，プロパティ，アセット参照，配置済み矩形を読み，PDF を出力します．


## 参照形式

Font Awesome は Markdown の画像記法から参照します．

```ss
let body = text <<
![home](fa-solid:house) Home
![github](fa-brands:github) Repository
>>
```

使える接頭辞は次の通りです．

| 接頭辞 | 意味 |
| --- | --- |
| `fa:` | 既定の Font Awesome 参照 |
| `fas:` | solid |
| `far:` | regular |
| `fab:` | brands |
| `fa-solid:` | solid |
| `fa-regular:` | regular |
| `fa-brands:` | brands |

アイコンが見つからない場合は，通常の画像と同じように描画できません．名前と接頭辞を確認します．


## 入力

描画器が読む主な値は次の通りです．

| 値 | 用途 |
| --- | --- |
| `content` | 本文，コード，数式，アセットパス |
| `role` | object の意味上の分類 |
| `payload` | 描画時のペイロード種別 |
| `props` | 文字，色，折り返し，描画方式 |
| `bbox` | 配置済み矩形 |

## 生成物

`ss render` は PDF を出力します．中間ファイルやキャッシュは `.ss-cache` 以下に置きます．

```sh
ss render slide.ss .ss-cache/out.pdf
```

公開用の PDF だけをソース管理へ含めるかどうかは，プロジェクトの運用で決めます．通常の開発では，生成物を `.ss-cache` に置きます．

## 診断

描画時には，外部コマンドの失敗，アセットの読み込み失敗，数式のコンパイル失敗，object のはみ出し，フォントやアイコンの解決失敗が分かります．

```sh
ss check slide.ss
ss render slide.ss .ss-cache/out.pdf
```

`check` と `render` は目的が違います．`check` は描画前に分かる誤りを確認します．`render` は実際の PDF 生成に必要な処理まで実行します．
