# ss documentation writing plan

このメモは，章立てと執筆状況を管理するためのものです．公開ページには出さないため，ファイル名を `_` で始めています．

## 方針

- `ss` を全く知らない人が，この文書だけで最初の PDF を作り，既存のコンポーネントを使い，配置を調整し，必要なら自分のコンポーネントを定義できる状態にする．
- 文章は淡々と説明する．不要な英単語を増やさない．専門用語は必要な場所で使い，最初に使う場面で説明する．
- 飾った対比表現を避ける．主張を飾らず，できること，書く場所，結果を説明する．
- すべてのページは日本語と英語を対応させる．日本語側を先に整え，英語側は同じ構成で説明する．
- コード例は実際の構文に合わせる．未実装の機能を確定した仕様のように書かない．
- Getting Started は長くしてよい．最初の導線で困らないことを優先する．
